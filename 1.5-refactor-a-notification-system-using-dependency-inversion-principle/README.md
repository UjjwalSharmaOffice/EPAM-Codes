# Task : Refactor a Notification System using Dependency Inversion Principle
## Description : 
Design a notification system with a high-level 'NotificationService' class depending on an abstraction ('INotificationSender'), not on concrete implementations like 'EmailSender' or 'SmsSender'.
## Topics Covered : 
Dependency Inversion Principle
## Objective : 
Learner will learn how to refactor a system to follow the Dependency Inversion Principle by depending on abstractions rather than concrete implementations.
